## 요약

보안뉴스에 따르면, 공격자들은 마이크로소프트 Teams와 같은 신뢰받는 협업 플랫폼을 이용해 랜섬웨어 공격을 은폐하고, 정상 트래픽처럼 위장한 채 내부 네트워크에서 장기간 활동한 것으로 보입니다.

핵심 포인트는 단순히 "악성코드가 발견되지 않았다"는 것이 아니라, 공격자가 조직이 이미 신뢰하는 SaaS 서비스와 인증 흐름을 활용해 탐지 회피를 시도했다는 점입니다. 특히 악성 트래픽을 정상 서비스처럼 보이게 만들고, 내부 보안 도구를 무력화하는 공격 수법이 주요 특징입니다.

이번 사례는 단순한 악성코드 유포가 아니라, 신뢰 기반 인프라와 제어 체계가 얼마나 쉽게 악용될 수 있는지를 보여줍니다.

## 공격 흐름 요약

1. 초기 침투 단계에서 공격자는 SQL 서버 취약점이나 기타 취약한 경로를 통해 내부 네트워크에 접근했습니다.
2. 이후 정상적인 협업 툴의 트래픽을 이용해 명령 및 제어(C2) 통신을 수행했습니다.
3. DLL 사이드로딩, 무단 계정 생성, 방화벽 규칙 수정 등으로 탐지를 회피했습니다.
4. 보안 도구를 종료하거나 우회하는 드라이버 기반 기법을 활용해 장기적인 접근을 확보했습니다.
5. 최종적으로 랜섬웨어를 실행하거나 추가 페이로드를 배포했습니다.

## OWASP Top 10 관점 분석

### A01: Broken Access Control

공격자가 내부 네트워크와 서비스 내에서 불필요하거나 과도한 권한을 확보한 가능성이 높습니다. 특히 계정 탈취, 권한 남용, 내부 시스템의 과도한 접근 통제가 이 사례에서 핵심 문제로 보입니다.

- 사용자 계정과 서비스 계정의 접근 범위가 과도하게 넓을 경우 피해가 커집니다.
- 내부 네트워크에서의 권한 분리가 충분하지 않으면 공격자가 빠르게 이동할 수 있습니다.

### A04: Insecure Design

Teams 같은 신뢰할 수 있는 서비스가 공격 경로로 악용되는 구조는 설계 단계에서 보안 위협을 충분히 고려하지 못했을 가능성을 시사합니다.

- 조직은 협업 툴을 "신뢰 가능한 경로"로 간주하지만, 실제로는 공격자에게도 활용될 수 있습니다.
- 서비스의 신뢰성만으로 인프라를 보호하려는 설계는 위험합니다.

### A05: Security Misconfiguration

공격자가 방화벽 규칙, 모니터링 설정, 내부 계정 정책을 조정한 점은 보안 설정이 적절히 관리되지 않았음을 보여줍니다.

- 화이트리스트 기반 통제가 잘못 구성되면 정상 트래픽과 악성 트래픽을 구분하기 어렵습니다.
- 네트워크 정책과 EDR/로그 설정이 일관되게 적용되지 않으면 탐지가 늦어집니다.

### A06: Vulnerable and Outdated Components

이번 공격에서 SQL 서버 취약점, 취약한 드라이버, 무기화된 DLL 등 오래되거나 관리되지 않는 구성 요소가 활용된 것으로 보입니다.

- 보안 업데이트가 지연된 시스템은 공격자의 초기 침투 지점이 됩니다.
- 드라이버나 커널 수준의 구성 요소는 특히 높은 위험을 가집니다.

### A07: Identification and Authentication Failures

공격자가 추가 계정을 생성하거나 인증 흐름을 악용한 점은 인증 체계의 취약점과 연결됩니다.

- 다단계 인증이 적용되지 않거나 세션 관리가 약하면 피해가 확대됩니다.
- 신뢰된 SaaS 서비스의 세션을 악용하면 일반적인 네트워크 경보를 우회하기 쉽습니다.

### A08: Software and Data Integrity Failures

DLL 사이드로딩, 무기화된 실행 파일, 신뢰 가능한 프로세스 주입은 데이터와 소프트웨어 무결성 문제와 직접 관련됩니다.

- 정상 실행 파일처럼 보이는 악성 구성 요소는 검증 체계의 한계를 드러냅니다.
- 코드 서명, 파일 무결성 검사, 실행 정책이 약하면 공격이 유효해집니다.

### A09: Security Logging and Monitoring Failures

가장 중요한 교훈 중 하나는, 정상적인 서비스 트래픽을 악성 활동으로 분별하지 못한 보안 모니터링 실패입니다.

- 협업 플랫폼의 아웃바운드 통신을 실시간으로 파악하지 못하면 대응이 늦습니다.
- 행동 기반 탐지와 엔드포인트 이벤트 분석이 보완되어야 합니다.

## 결론

이번 공격은 단순한 랜섬웨어 사건이 아니라, "신뢰받는 서비스가 곧 안전한 서비스라는 가정"을 무너뜨린 사례입니다. 특히 OWASP Top 10 기준으로 보면 A01, A04, A05, A06, A07, A08, A09가 모두 연결되는 복합적 공격으로 볼 수 있습니다.

기업은 단순히 방화벽과 EDR만으로 대응하기보다, SaaS 서비스의 사용 흐름, 인증 체계, 내부 권한 분리, 모니터링 체계를 함께 재설계해야 합니다. 특히 협업 플랫폼을 포함한 신뢰 경로에 대한 검증은 이제 기본적인 보안 준비로 봐야 합니다.

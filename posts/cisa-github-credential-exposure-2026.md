## 요약

2026-05-20 보안뉴스 보도에 따르면, 미국 사이버보안 및 기반시설안보국(CISA) 협력업체인 `Nightwing` 소속 관리자가 민감한 정부 기밀 클라우드 자격증명을 공개 GitHub 저장소에 노출시킨 것으로 확인됐다.

- 저장소 이름은 `Private-CISA`로, 2025년 11월부터 2026년 5월 중순까지 공개 상태로 방치
- 노출된 자격증명에는 AWS GovCloud 최고 관리자 자격증, 내부 시스템 로그인 정보, Artifactory 자격증명, DevSecOps 환경 비밀번호 등이 포함
- GitGuardian가 최초로 탐지했고, CISA는 26시간 만에 저장소를 비공개 처리했으나 해당 자격증명은 이후에도 48시간 동안 유효 상태로 남아 있었다
- 관리자는 GitHub의 자동 비밀탐지 기능을 고의로 비활성화한 뒤 파일을 업로드했으며, 개인 장비를 업무용 동기화 용도로 사용한 정황도 드러났다

## 분석

이번 사고는 단순한 자격증명 유출을 넘어, 고신뢰 환경에서의 인적 오류와 잘못된 비밀관리 관행이 결합해 얼마나 큰 위험을 만드는지를 보여준다.

### 1. 공개 저장소에 비밀을 두는 행위

공용 GitHub 저장소는 본질적으로 접근이 제한되지 않는다. 민감한 클라우드 키, 내부 시스템 계정, Artifactory 자격증명을 그대로 업로드했다는 것은 기본적인 보안 원칙(`Never commit secrets`)이 완전히 무시된 사례다.

### 2. 비밀 탐지 및 차단 시스템 우회

GitHub의 비밀번호/키 외부 노출 자동 차단 기능을 비활성화한 점은 탐지 보완과 정책 강화를 의미한다. 조직은 탐지 시스템의 무력화 시도를 감지하고 경고하는 추가 통제 장치가 필요하다.

### 3. 키 회전과 유효성 관리 실패

문제가 보고된 후에도 핵심 인증키가 48시간 동안 유효했던 것은 대응 절차의 허점이다. 고위험 자격증명이 노출된 경우, 즉시 회전 및 무효화 절차를 수행해야 한다.

### 4. 공급망 및 인프라 악용 가능성

AWS GovCloud 관리자 권한과 Artifactory 자격증명은 단순한 데이터 유출을 넘어서, 악성 코드 배포, 소프트웨어 공급망 타깃화, 내부 네트워크 접근으로 이어질 수 있다. 특히 정부 클라우드 환경에서의 권한 도용은 심각한 국가 인프라 위험이다.

### 5. 인적 요소와 조직 환경

보도에 따르면 예산 감축과 조직 개편 압박이 인적 오류의 요인이 될 수 있다는 지적도 있었다. 보안은 기술적 통제뿐 아니라, 규정 준수와 업무 환경 개선을 함께 고려해야 한다.

## 권고

1. 공개 저장소에 대한 비밀 스캐닝과 업로드 차단을 기본 정책으로 적용
2. GitHub 및 내부 저장소의 로깅·모니터링 강화, 탐지 우회 행동에 대한 경고 체계 마련
3. 민감한 자격증명은 비밀관리를 위한 전용 시스템(예: HashiCorp Vault, AWS Secrets Manager)으로 이전
4. 유효성 검사/회전 자동화: 노출 감지 시 즉시 키 무효화와 재발급 프로세스 실행
5. 개인 장비 사용 규제 및 업무용 데이터 동기화 정책 강화

## 결론

이번 CISA 협력업체 GitHub 노출 사고는 `기술적 보안`과 `인간·운영 보안`의 결합 실패 사례다. 조직은 자격증명 관리의 모든 단계를 점검하고, 공개 저장소를 통한 노출을 물리적으로 차단하는 방향으로 보안 태세를 조정해야 한다.
